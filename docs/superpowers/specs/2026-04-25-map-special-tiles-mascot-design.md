# 扫雷游戏增强设计文档

**日期：** 2026-04-25  
**范围：** 选关地图页面 / 特殊格子惊喜机制 / 角色动画表情系统

---

## 一、选关地图页面（Adventure Map Screen）

### 概述

在现有游戏界面前增加独立的选关页面。游戏启动时默认显示地图页，选择关卡后切换至游戏界面，游戏界面内保留"返回地图"按钮。

### 页面流程

```
启动 → 地图页 → 点击已解锁关卡 → 游戏页
                                        ↓
                              胜利 → 解锁动画 → 返回地图页
                              失败（生命归零）→ 返回地图页
```

### 五个关卡场景

| 关卡 | 场景 | 格子数 | 地雷数 |
|------|------|--------|--------|
| 1 | 🌲 森林 | 6×6 | 4 |
| 2 | 🏜️ 沙漠 | 8×8 | 10 |
| 3 | 🌊 海洋 | 9×9 | 15 |
| 4 | ⛰️ 雪山 | 10×10 | 22 |
| 5 | 🚀 太空 | 10×10 | 30 |

### 视觉设计

- **背景：** 羊皮纸卷轴感米黄色背景（`#f5e6c8`），带轻微纹理
- **关卡卡片：** 圆形场景插画（SVG），卡片有阴影，鼠标悬停放大 1.08 倍
- **锁定状态：** 灰色遮罩 + 锁图标覆盖在卡片上
- **解锁动画：** 完成关卡后回到地图页，对应卡片播放锁碎裂 + 金色闪光动画（0.6s CSS keyframes）
- **星级显示：** 每张卡片底部显示 0~3 颗星（取历史最佳）

### 解锁规则

- 第 1 关默认解锁
- 完成任意星级即解锁下一关（不要求完美通关）
- 解锁进度存 `localStorage`，key：`unlockedLevel`（值为已解锁的最高关卡编号，默认 `1`）

### 状态管理

新增全局变量 `currentScreen`（`'map'` | `'game'`），通过切换对应 `div` 的 `display` 实现页面切换，无路由。

## 二、特殊格子 & 惊喜机制（Special Tiles）

### 三种特殊格子

| 类型 | 图标 | 效果 | 背面发光颜色 |
|------|------|------|-------------|
| 幸运星 `star` | ⭐ | 揭开得双倍积分 | 金色 `#ffd700` |
| 宝箱 `chest` | 💎 | 得 +1 提示次数 | 蓝色 `#4fc3f7` |
| 加速器 `boost` | ⚡ | 计时器暂停 5 秒 | 紫色 `#ce93d8` |

### 放置规则

- 每局在首次点击后（与地雷同时）随机放置于安全格中
- 数量：关卡 1-2 放 2 个，关卡 3-5 放 3 个
- 优先放置在数字格（`adjacentMines > 0` 的安全格），确保玩家主动揭开才能触发，避免被洪水填充自动揭开；若数字格不足则放在任意安全格
- 数据结构：`board[r][c].special = 'star' | 'chest' | 'boost' | null`

### 背面提示效果

- CSS `box-shadow` 脉冲动画（`@keyframes specialPulse`），2s 循环
- 幸运星：金色光晕；宝箱：蓝色光晕；加速器：紫色光晕
- 发光强度克制（`box-shadow: 0 0 8px 2px color`），不在视觉上压过正常格子

### 揭开效果

1. 格子先做 `scale(1.2)` 弹跳（0.3s）再落回
2. 显示对应图标（大字居中，0.4s fade-in）
3. 触发对应功能效果
4. 顶部 Toast 提示（中英文 i18n）

### 计时暂停实现

加速器揭开后：`clearInterval(state.timerInterval)`，5 秒后重新 `startTimer()`，期间在计时器位置叠加倒计时光环动画。

## 三、角色动画 & 表情系统（Mascot Expressions）

### 猫咪（Cat）四种状态

| 状态 CSS 类 | 触发条件 | 动画描述 |
|------------|----------|---------|
| `mascot--happy` | 胜利 / 连击 ≥3 | 身体弹跳，眼睛变弧形，头顶冒金色星星粒子 |
| `mascot--scared` | 踩雷 / 失去生命 | 向后跳，眼睛变圆，冒汗珠（3个小圆点飞出）|
| `mascot--cheer` | 连击 ≥6 | 挥动爪子加速，身体微微发光 |
| `mascot--cry` | 游戏失败（生命归零）| 眼泪流下，耳朵下垂（`transform: rotate`）|

默认状态为无额外类，保持现有温和 bob 动画。

### 乌龟（Turtle）同步反应

| 触发条件 | 乌龟动作 |
|----------|---------|
| 胜利 | 从壳里探头，眼睛变弯弯弧形 |
| 踩雷 | 缩回壳里，只露眼睛 |
| 失败 | 壳上出现裂缝（CSS `::after` 绘制） |

### 点击互动

- 点击猫咪 SVG 元素触发
- 随机显示鼓励气泡（8条，中英文各一套），气泡淡入 0.2s，停留 2s，淡出 0.3s
- 气泡出现在猫咪头顶，带尾巴三角
- 冷却：同一条不连续重复（shuffle 数组轮播）

**中文鼓励语（8条）：**
1. 加油！你一定行的！
2. 仔细想想，答案就在眼前！
3. 好棒！继续保持！
4. 没关系，再试一次！
5. 你是最厉害的小探险家！
6. 相信自己！
7. 慢慢来，不着急～
8. 耶！离胜利不远了！

### 自动互动

- 连续 30 秒无任何操作：猫咪自动说一句鼓励语（防止小朋友卡住），同时轻微挥爪

### 连击联动

- 连击数 ≥3：进入 `mascot--happy` 状态
- 连击数 ≥6：额外加 `mascot--cheer` 类，猫咪轮廓开始发彩虹光

### 实现方式

- 纯 CSS 动画类切换，SVG 内部元素（眼睛路径、爪子、眼泪）用 `transform` / `opacity` keyframes 控制
- JS 只负责添加/移除 CSS 类，状态变量 `mascotMood` 记录当前表情
- 每次状态切换前先移除所有表情类，再添加新类

## 四、技术影响与兼容性

- **无新依赖**：全部实现基于原生 HTML/CSS/JS，延续单文件架构
- **localStorage 新增 key：** `unlockedLevel`（数字），已有 key（`bestScore_*`、`achievements`、`lang`）不变
- **i18n 扩展：** 新增 `specialTile_star`、`specialTile_chest`、`specialTile_boost`、`mapTitle` 等 key，中英文各一套
- **性能：** 特殊格子 CSS 脉冲动画只在格子未揭开时运行，揭开后移除动画类，无持续性能损耗
